# ТЗ — Лендинг Hunar

**Продукт:** Hunar — маркетплейс найма для IT-рынка Узбекистана
**Страница:** `/` (главный лендинг)
**Версия:** 1.0 (MVP)

---

## 1. Цель лендинга

Объяснить за 5 секунд, что это и кому полезно, и развести два потока пользователей по двум действиям: **«Ищу работу»** и **«Нанимаю»**. Вторичная цель — органический трафик: лендинг и каталог вакансий должны индексироваться.

**Главная метрика:** доля посетителей, нажавших один из двух основных CTA (регистрация кандидата / переход для работодателя).

## 2. Аудитория

Двусторонняя, на одной странице:
- **Кандидаты** — IT-специалисты Узбекистана (mid/senior как ядро beachhead).
- **Работодатели** — IT-компании, стартапы, резиденты IT Park.

Тон: профессиональный, без «инфоцыганщины», конкретный. Ключевые сообщения — про **качество и курацию** (в отличие от спама на других площадках) и **прозрачность зарплат в UZS и USD**.

## 3. Структура секций (сверху вниз)

1. **Hero**
   - Заголовок, бьющий в позиционирование (напр. «IT-работа в Узбекистане без спама» / «Прозрачные зарплаты в сумах и долларах»).
   - Подзаголовок (1–2 строки про суть).
   - Поиск-бар (по навыку/должности) → ведёт в `/jobs`.
   - Два CTA: «Найти работу» и «Разместить вакансию».

2. **Ценность / «почему мы»** — 3–4 карточки:
   - Курация: каждая вакансия проверена, без спама.
   - Прозрачность: зарплата в UZS и USD сразу.
   - Верификация навыков (бейдж).
   - Фокус на IT / remote-готовность.

3. **Как это работает** — 3 шага для кандидата + 3 шага для работодателя (две колонки/табы).

4. **Свежие вакансии** — 6–8 карточек последних одобренных вакансий (динамика из БД), кнопка «Все вакансии» → `/jobs`.

5. **Блок «Для работодателей»** — короткий оффер + CTA → `/for-employers`.

6. **FAQ** — 4–6 вопросов (бесплатно ли для кандидата, как защищены данные/CV, чем отличаетесь от других, как происходит контакт с работодателем).

7. **Финальный CTA** — повтор двух действий.

8. **Footer** — навигация, ссылки на `/privacy`, `/terms`, `/consent`, контакты, язык (RU/UZ).

## 4. Копирайт (гайд)

- Язык интерфейса: RU + UZ (переключатель).
- Зарплаты везде — в формате **UZS + USD**.
- Избегать общих фраз («лучшая платформа»). Говорить конкретикой: «вакансия проверена модератором», «контакт работодателя после одобрения отклика».
- Не обещать того, чего нет в MVP (нет чата — не писать «общайтесь в чате»; писать «получите контакт работодателя»).

## 5. Анимации (Motion / framer-motion)

Сдержанно, не в ущерб скорости:
- Плавное появление секций при скролле (fade/slide-in, небольшой stagger).
- Микро-ховеры на карточках вакансий и кнопках.
- Анимация hero — лёгкая, без тяжёлых фоновых эффектов.
- **Запрет:** ничего, что блокирует первый рендер или роняет производительность на мобильных. Уважать `prefers-reduced-motion`.

## 6. SEO

- SSR/SSG, быстрый первый рендер.
- `<title>`, `meta description`, OpenGraph/Twitter-карточки.
- Семантическая разметка, `h1` один на странице.
- Блок «Свежие вакансии» — реальные ссылки на `/jobs/[slug]` (внутренняя перелинковка).
- `sitemap.xml`, `robots.txt`.

## 7. Адаптивность

- **Mobile-first.** Основной трафик — мобильный.
- Hero и поиск-бар удобны на узком экране; CTA крупные, тапабельные.
- Карточки вакансий перестраиваются в одну колонку.

## 8. Техническая реализация

- Next.js (App Router), компонент страницы — серверный; интерактив (поиск-бар, табы, ховеры) — клиентские островки.
- Свежие вакансии тянутся серверно из Supabase (`jobs` со `status = approved`, последние N).
- Tailwind для вёрстки, Motion для анимаций.
- Изображения (лого в карточках) — оптимизированные, через next/image.

## 9. Критерии приёмки

- [ ] Два основных CTA видны без скролла на мобильном и десктопе.
- [ ] Поиск-бар ведёт в `/jobs` с применённым запросом.
- [ ] Секция «Свежие вакансии» показывает реальные одобренные вакансии из БД.
- [ ] Страница индексируется, есть meta/OG, sitemap.
- [ ] Анимации не ломают производительность, уважают `prefers-reduced-motion`.
- [ ] Переключатель RU/UZ работает.
- [ ] Footer содержит ссылки на юридические страницы.
